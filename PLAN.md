# Reovim 移植プロジェクト詳細計画 (PLAN.md)

本ドキュメントは、NeovimのCコードをRustへ段階的に移植するための各フェーズの具体的な実装計画を定めたものです。

---

## 📋 フェーズ 1: ビルドシステムの統合 (CMake + Cargo)
NeovimのCMakeビルドプロセスの中で、Rustプロジェクト (`reovim`) を自動的にコンパイルし、生成されたスタティックライブラリ (`.a`) を Neovim の実行バイナリにリンクして呼び出せるようにします。

### 📌 タスクリスト
- [ ] **1.1: Rustプロジェクトを `staticlib` として構成する**
  - `Cargo.toml` に `[lib]` セクションを追加し、`crate-type = ["staticlib"]` を設定。
  - `src/lib.rs` を新規作成し、Cから呼び出せるように `#[no_mangle] pub extern "C" fn` を使用したダミー関数（例: `reovim_hello`）を定義。
- [ ] **1.2: Neovimの CMakeLists.txt の修正**
  - CMakeから `cargo build` を自動で呼び出すカスタムターゲットを追加。
  - 生成された Rust のライブラリ (`libreovim.a` 等) を Neovim 実行バイナリのリンク対象に追加。
  - macOS (Apple Silicon/Intel) や各環境で適切にスタティックライブラリが解決されるようにパスとリンカフラグを調整。
- [ ] **1.3: Cコードからの呼び出し疎通確認**
  - `vim_src/neovim-0.12.2/src/nvim/main.c`（もしくは初期化処理のファイル）に Rust 関数のプロトタイプ宣言を追加。
  - 起動シーケンスの途中で Rust 関数を呼び出し、標準出力やログファイルへの出力を確認する。

---

## 📦 フェーズ 2: 依存の少ないユーティリティモジュールの移植
Cコードベースの中で、他のモジュールへの依存（グローバル変数や構造体の参照）が極めて少なく、移植しやすい「末端（Leaf）モジュール」から順に Rust に書き換えます。

### 📌 移植対象候補と手順
- [ ] **2.1: `sha256.c` の移植**
  - `vim_src/neovim-0.12.2/src/nvim/sha256.c` のロジックを確認。
  - Rustの外部クレート（`sha2` 等）を使うか、または同等ロジックをピュアRustで再実装。
  - C側の関数インターフェース（例: `sha256_key`, `sha256_self_test`）を Rust で FFI 経由で公開。
  - C側の `sha256.c` を CMake のビルド対象から外し、代わりに Rust の実装にリンクさせる。
- [ ] **2.2: `path.c`（パス操作ヘルパー）の移植**
  - 文字列やパス判定（`path_is_absolute` 等）を行う関数群を Rust の `std::path::Path` や文字列操作で書き換える。
- [ ] **2.3: `map_glyph_cache.c`（グリフキャッシュ）の移植**
  - UIに関連する自己完結したメモリキャッシュ機構を Rust のデータ構造で再実装。

### 🧪 検証方法
- 移植を行うたびに、Neovimの既存のユニットテスト・機能テスト（`make functionaltest` 等）を実行し、動作のデグレがないことを保証する。

---

## 🔗 フェーズ 3: FFI バインディングの自動生成とデータ構造の共有
Rust側からC側のより複雑なデータ構造（例：ウィンドウを表す `win_T`、バッファを表す `buf_T` など）を操作できるようにするため、双方向の FFI 環境を整備します。

### 📌 タスクリスト
- [ ] **3.1: `bindgen` の導入**
  - Rustの `build.rs` または外部ビルドスクリプトとして `bindgen` を構成。
  - Neovim のCヘッダーファイルを解析し、自動的にRust用の構造体や関数のバインディングを自動生成する。
- [ ] **3.2: 構造体操作の安全なラッパー作成**
  - `bindgen` で生成された `unsafe` なC構造体アクセスを、Rustらしく安全に扱うためのスマートポインタやラッパー型を設計。

---

## 🚀 フェーズ 4: コアロジックの段階的リプレース
ユーティリティや周辺のデータ構造の移植が完了した後、Neovimのエディタエンジン本体に踏み込んで Rust 化を進めます。

### 📌 対象エリア
- [ ] **4.1: MessagePack-RPC 関連の移植**
  - プロトコルやシリアライザ周りを Rust の堅牢な型システムと `serde` 等で置き換える。
- [ ] **4.2: TUI / UI ブリッジ**
  - 端末描画やイベントループの Rust 化。
- [ ] **4.3: Lua API / 評価器との連携**
  - NeovimのLuaインテグレーション部分と Rust とのインターフェースの整理。
